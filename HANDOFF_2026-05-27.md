## Critical Facts a New Session Must Know

### 1. The paper is about FULL external shock transmission — both channels
- **Channel 1 (monetary):** US FFR → HK HIBOR → property prices (LERS peg)
- **Channel 2 (trade):** China GDP → HK exports → HK GDP

Do NOT narrow focus to just property prices. Both channels are headline findings.

**Why I(1) in unemployment/cpi doesn't kill BVAR:**
- Sims, Stock, Watson (1990): VAR in levels OK for inference on stationary regressors even with I(1) in the system
- Minnesota prior with ar_coefficients=0.95 partially absorbs near-unit-root behavior
- Δu robustness confirmed: headline IRFs unchanged (Lesson 23)

### 4. Johansen result — VECM not warranted (Lesson 24)

Endogenous-only Johansen (unemployment, cpi_inflation, hk_property_price_idx):

| Test | Stats | 95% CV | **Rank** |
|---|---|---|---|
| Trace | [24.57, 9.56, 0.75] | [29.8, 15.49, 3.84] | **0** |
| Max-eigenvalue | [15.0, 8.82, 0.75] | [21.13, 14.26, 3.84] | **0** |

**VECM cannot be estimated — rank=0 means no cointegrating relationship.**

Paper sentence:
> "Johansen trace and max-eigenvalue tests on the confirmed I(1) endogenous variables (unemployment, CPI inflation, property price index) find no cointegrating relationship at the 95% level (rank=0, both tests). VECM estimation is therefore not warranted. This finding supports the BVAR-in-levels specification as the appropriate framework for this system."

### 5. hk_var_model.py Johansen is WRONG — never use it

The pipeline in `hk_var_model.py` runs Johansen on ALL 7 variables including `us_ffr` and `china_gdp` (exogenous). This is a specification error. It produced spurious rank=1 with FFR in the cointegrating vector. The HIBOR-FFR co-movement is LERS peg mechanics (institutional transmission), not equilibrium cointegration.

**Always use manual `statsmodels.tsa.vector_ar.vecm.coint_johansen` on endogenous I(1) subset only.**

## Alexandria API — Critical Reference

```python
from alexandria import MinnesotaBayesianVar

bvar = MinnesotaBayesianVar(
    endogenous=df[endog_cols].values,
    exogenous=df[exog_cols].values,
    lags=4, pi1=0.085, pi2=1.0, pi3=1.0,
    credibility_level=0.90, iterations=5000
)
bvar.estimate()

# IRF — returns shape (6, 6, 9, 3) = [response, impulse, horizon, band(lower/med/upper)]
irf, _ = bvar.impulse_response_function(h=9, credibility_level=0.90)

# Access: irf[response_idx, impulse_idx, horizon_idx, band_idx]
# Variable order: 0=exports, 1=gdp, 2=cpi, 3=unemployment, 4=hibor, 5=property
# band: 0=lower, 1=median, 2=upper
```

**Method is `.impulse_response_function()` NOT `.irf()` — `.irf()` does not exist.**
