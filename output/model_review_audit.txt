========================================
CRITICAL — Issue #1
========================================
FORMULA:  Forecast output for differenced variables
EVIDENCE: Variables 'unemployment' and 'us_ffr' are first-differenced before
          estimation. The VAR therefore models d(unemp) and d(ffr).
          But forecast_scenarios() and _plot_scenarios() present ALL variables
          as if they are in LEVELS.
          Line 808: ax.plot(hist.index, hist.values, ...) — plots d(unemp)
          Label says "HK Unemployment Rate (%)" — implies level.
          The scenario chart shows d(unemp) ~ 0.1 and the user reads
          "unemployment is 0.1%" when it actually means "unemployment
          changed by 0.1pp this quarter."
          The forecast CSV likewise has no transform indicator.

IMPACT:   Any policy interpretation of unemployment or interest rate forecasts
          is WRONG. A user reading the chart would conclude unemployment is
          near zero when it is actually ~5%.

CORRECTED FORMULATION:
  1. Maintain a transforms registry mapping variable -> transform applied.
  2. After forecasting in transformed space, INVERT transforms:
       if transform == "first_diff":
           level_forecast[t] = last_known_level + cumsum(diff_forecast[0:t])
  3. Plot and export forecasts in LEVELS, with clear annotation.
  4. Store both level and transformed forecasts in output.

HIGH — Issue #2
========================================
FORMULA:  Scenario shock specification
EVIDENCE: In forecast_scenarios() line 770-775:
            shock1[-1, china_idx] -= 3.0    (china_gdp is in LEVELS)
            shock1[-1, ffr_idx]   += 1.5    (us_ffr is FIRST-DIFFERENCED)
          The user intends "FFR rises 1.5pp" but what happens is "the quarterly
          CHANGE in FFR is 1.5pp larger than baseline."
          If FFR was already falling by 0.5pp, the shocked change is +1.0pp,
          not +1.5pp in the level.

IMPACT:   Scenario magnitudes are ambiguous; external demand and rate shocks
          are not comparable in economic terms.

CORRECTED FORMULATION:
  - Define all shocks in LEVEL space: "China GDP growth falls to X%",
    "FFR rises to Y%."
  - Convert level shocks to transformed-space shocks:
       if transform == "first_diff":
           transformed_shock = desired_level - last_known_level
       else:
           transformed_shock = desired_level


========================================
HIGH — Issue #3
========================================
FORMULA:  ADF stationarity test under structural breaks
EVIDENCE: The ADF test assumes no structural breaks under H0 (unit root).
          Perron (1989) showed that structural breaks bias ADF toward
          non-rejection — i.e., finding a unit root when the series is
          actually stationary with a break.
          Unemployment rate may be stationary around a shifted mean (GFC,
          COVID) but ADF finds it non-stationary (p=0.258).
          Over-differencing a stationary process LOSES the level information
          and increases forecast error variance.

IMPACT:   Potential misspecification of the estimation system. If unemployment
          is actually I(0) with level breaks, differencing is inappropriate
          and the model loses the equilibrium-reversion property.

CORRECTED FORMULATION:
  - Add KPSS test (null = stationarity) as a confirmatory second opinion.
  - If ADF says non-stationary but KPSS says stationary, flag a conflict
    and default to LEVEL (conservative approach).
  - Optionally add Zivot-Andrews break-point unit-root test.


========================================
HIGH — Issue #4
========================================
FORMULA:  IRF computation inconsistency between VAR and BVAR
EVIDENCE: statsmodels VAR result.irf(periods) returns orthogonalized
          (Cholesky) IRFs by default.
          The BVAR fallback _plot_irf_from_coefs() computes REDUCED-FORM
          IRFs: one-unit shocks to each variable separately (line 579-585).
          Reduced-form shocks are correlated — a shock to variable 1 is
          simultaneously a partial shock to all others via Sigma_u.
          Cholesky shocks are orthogonalized — a shock to variable 1 has
          zero contemporaneous effect on variables ordered after it.

IMPACT:   IRF charts are NOT comparable between VAR and BVAR runs.
          Economic interpretations change depending on model choice.

CORRECTED FORMULATION:
  - Compute Cholesky decomposition of Sigma_u: P = chol(Sigma_u)
  - Structural IRF at horizon h: Theta_h = Phi_h @ P
    where Phi_h is the h-step MA coefficient from the companion form.
  - Apply Cholesky in BOTH paths for consistency.


========================================
HIGH — Issue #5
========================================
FORMULA:  No cointegration testing before differencing
EVIDENCE: If some I(1) variables share a long-run equilibrium (cointegration),
          a VAR in first differences is MISSPECIFIED — it discards the
          error-correction term. Engle-Granger (1987), Johansen (1991).
          For Hong Kong: unemployment and GDP growth plausibly cointegrate
          (Okun's law); HIBOR and FFR cointegrate (currency board peg).
          A VECM would preserve these long-run relationships while modeling
          short-run dynamics.

IMPACT:   The differenced VAR may produce inferior long-horizon forecasts
          because it loses equilibrium-reversion forces.

CORRECTED FORMULATION:
  - Before differencing, run Johansen trace/max-eigenvalue test on the
    I(1) subset.
  - If cointegration rank r > 0, report it and note in the diagnostics
    that a VECM may be preferred for longer horizons.
  - For the current pipeline, at minimum flag this in the methods note.


========================================
MEDIUM — Issue #6
========================================
FORMULA:  BVAR is scalar ridge, not Minnesota prior
EVIDENCE: The code labels fit_bvar_ridge() as "BVAR" but implements:
            B_hat = (X'X + lambda * I)^{-1} X'Y
          A true Minnesota prior (Litterman 1986, Doan/Litterman/Sims 1984)
          has VARIABLE-SPECIFIC, LAG-SPECIFIC shrinkage:
            Var(A_{ij}^{(l)}) = (lambda_1 / l^{lambda_3})^2   if i==j
            Var(A_{ij}^{(l)}) = (lambda_1*lambda_2*s_i / (l^{lambda_3}*s_j))^2  if i!=j
          Key missing features:
            a) Own-lag vs cross-lag differentiation (lambda_2 << 1)
            b) Lag decay (l^{lambda_3} shrinks distant lags more)
            c) Scale normalization (s_i / s_j)

IMPACT:   Scalar ridge over-shrinks own lags and under-shrinks cross lags
          relative to Minnesota. This reduces the BVAR's ability to capture
          the dominant own-persistence that characterizes macro data.

CORRECTED FORMULATION:
  - Implement Minnesota-style diagonal prior variance matrix V_prior:
      diag entry for A_{ij}^{(l)}: see formula above
  - Ridge becomes: B_hat = (X'X + V_prior^{-1})^{-1} X'Y

MEDIUM — Issue #7
========================================
FORMULA:  Bootstrap confidence bands
EVIDENCE: The residual bootstrap in forecast_scenarios() (line 750-756)
          resamples from the empirical residual pool but holds coefficients
          FIXED at their point estimates.
          True forecast uncertainty = innovation uncertainty + parameter
          uncertainty (Lütkepohl 2005, Ch. 3.5).
          With T~100 and 13 params per equation, parameter uncertainty is
          non-negligible.

IMPACT:   Confidence bands are too narrow — they understate the true
          forecast uncertainty, giving false precision.

CORRECTED FORMULATION:
  - Use a "pairs" or "wild" bootstrap that re-estimates the VAR on each
    replicate, OR
  - Add an analytic correction (asymptotic formula for forecast MSE
    including parameter uncertainty).

MEDIUM — Issue #8
========================================
FORMULA:  FRED real-data detection heuristic
EVIDENCE: Line 299: if col in df.columns and df[col].mean() > 50
          This decides whether GDP series are in index/level form (need
          pct_change) or already in growth rates.
          Failure mode: a GDP index with base year normalization could have
          mean < 50; a growth series in basis points could have mean > 50.

IMPACT:   Silent double-transformation or no-transformation of real data.

CORRECTED FORMULATION:
  - Use metadata from DATA_SPEC["target_transform"] to determine if a
    transform is needed, not the data values themselves.
  - Or: check if the series is monotonically increasing (index) vs
    mean-reverting (growth rate).


========================================
MEDIUM — Issue #9
========================================
FORMULA:  Synthetic DGP structural breaks
EVIDENCE: Breaks (GFC, COVID) are applied as additive post-hoc shifts to
          the VAR-simulated path (lines 207-222). This means:
            y_observed = y_VAR + break_shift
          But the estimation model assumes y_observed IS the VAR process.
          The break residuals are absorbed into u_t, inflating Sigma_u and
          biasing coefficient estimates.

IMPACT:   With synthetic data, the estimated VAR parameters do NOT match
          the true DGP parameters, even asymptotically.

CORRECTED FORMULATION:
  - Generate breaks as SHOCKS within the VAR simulation loop:
      for t in break_period:
          eps[t] += break_vector
  - This preserves the VAR structure and allows the model to "see" breaks
    as large but legitimate innovations.


========================================
LOW — Issue #10
========================================
FORMULA:  Residual covariance degrees of freedom
EVIDENCE: np.cov(resid, rowvar=False) uses denominator n-1.
          OLS VAR should use T - k*p - 1 (Lütkepohl convention) or T
          (ML convention). Difference is small for T~100.

CORRECTED: Use resid.T @ resid / (nobs - ncoef) for unbiased estimate.

LOW — Issue #11
========================================
FORMULA:  HIBOR proxy = FFR + N(0.3, 0.15)
EVIDENCE: Under the HK currency board, the HIBOR-FFR spread is time-varying
          and widens dramatically during stress (>100bp in 2008, 2019-2020).
          A constant-mean random spread misses the key dynamics.

CORRECTED: Use a regime-switching or GARCH-style spread, or at minimum
          widen the spread during known stress periods.
