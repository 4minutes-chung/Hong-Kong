REMAINING ISSUE #3 — LOW
========================================
FORMULA:  Global np.random.seed(42) at module load (line 55).
          All randomness shares a single seed. The bootstrap outcomes depend
          on how many random draws were consumed by earlier stages (data gen,
          backtesting). If the user changes --max-lags or --model-type, the
          bootstrap confidence bands shift even though the bootstrap logic is
          unchanged.
IMPACT:   Minor reproducibility fragility across configurations.
CORRECTED: Re-seed (or use a dedicated np.random.Generator) at the start of
           each stochastic block: data generation, backtesting, bootstrap.


========================================
REMAINING ISSUE #4 — LOW
========================================
FORMULA:  Cholesky ordering sensitivity.
          The IRF interpretation depends on variable ordering in the DataFrame.
          Current order: [gdp_growth, cpi_inflation, unemployment, hibor_3m,
          china_gdp, us_ffr]. Under Cholesky, the first variable is the most
          exogenous (no contemporaneous effect from others). Placing gdp_growth
          first implies it is not contemporaneously affected by any other
          variable — a strong assumption.
          Standard small-open-economy orderings often put external block first
          (us_ffr, china_gdp) then domestic (gdp, cpi, unemployment, hibor).
IMPACT:   IRF interpretation changes with ordering; current ordering is
          economically debatable.
CORRECTED: Document the ordering assumption explicitly. Optionally add a
           --cholesky-order CLI argument. Consider default:
           [us_ffr, china_gdp, gdp_growth, cpi_inflation, unemployment, hibor_3m]
           (external-first, domestic slow-to-fast ordering).
